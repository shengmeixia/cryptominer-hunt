Linux サーバー上の暗号通貨マイニングマルウェアおよび関連する不正プログラムを検出・調査・除去するための AI 駆動スキルです。

## なぜ必要か

暗号通貨マイニングマルウェアは、侵害された Linux サーバーに展開される最も一般的なペイロードの一つです。適切にクリーンアップするには、完全な攻撃チェーン ── マイナー、ドロッパー、永続化メカニズム、侵入経路、ラテラルムーブメント ── を数十もの潜伏場所にわたって追跡する必要があります。これは単調で、見落としが生じやすく、完全に実施することが困難です。

**cryptominer-hunt** は AI コーディングアシスタントをインシデントレスポンダーに変えます。固定的なチェックリストを与えるのではなく、*目標と原則*を定義し、各サーバー環境に合わせて AI が自律的にアプローチを調整します。人間のセキュリティアナリストのように調査を行います：証拠を追跡し、攻撃者の視点で考え、すべての手がかりを最後まで追います。

## 機能概要

| フェーズ | 目標 |
|----------|------|
| **トリアージ** | CPU 異常、不審なプロセス、異常なネットワーク活動、存在すべきでないファイルを特定 |
| **詳細調査** | 各指標について：バイナリの追跡、永続化のマッピング、ラテラルムーブメントの確認、システム整合性の検証、侵入経路の特定 |
| **レポート** | 構造化された報告 ── マルウェア一覧、永続化メカニズム、ネットワーク/ファイル IOC、攻撃チェーンの再構築、リスク評価 |
| **根絶** | プロセスの停止（ウォッチドッグを先に）、ファイルの削除、永続化の除去、C2 インフラのブロック |
| **検証** | すべての観点からシステムがクリーンであることを確認 |
| **推奨事項** | 認証情報のローテーション、パッチ適用、ラテラル監査対象、監視体制、再イメージングの要否判断 |

## インストール

スキルファイルを AI コーディングアシスタントのスキルディレクトリにコピーします：

```bash
# スキルディレクトリを作成
mkdir -p ~/.claude/skills/cryptominer-hunt

# スキルファイルをコピー
cp SKILL.md ~/.claude/skills/cryptominer-hunt/SKILL.md
```

または、このリポジトリを直接クローンします：

```bash
git clone https://github.com/shengmeixia/cryptominer-hunt.git ~/.claude/skills/cryptominer-hunt
```

## 使い方

Linux サーバーのターミナルセッションで：

```
# フルスキャン ── 調査とクリーンアップ（破壊的な操作の前に確認を求めます）
/cryptominer-hunt

# ドライラン ── 調査とレポートのみ、変更は行いません
/cryptominer-hunt --dry-run
```

### 使用場面

- サーバーの CPU 使用率が原因不明で高い
- 侵害の疑いがある、または確認されている
- インシデント対応後、クリーンアップが完全かどうかを確認したい
- インターネットに面したサーバーの定期セキュリティ監査

## 設計思想

このスキルは意図的に**目標指向型であり、手順規定型ではありません**。*何を*調査するかを定義し、*どのように*調査するかは定義しません。

- **ハードコードされたコマンドはゼロ。** AI が OS と環境に応じて適切なツールを選択します。
- **証拠駆動型。** 各発見がより深い調査のトリガーとなります ── マイナーの発見はドロッパーの存在を、ドロッパーは侵入経路を、侵入経路は永続化メカニズムの存在を示唆します。
- **攻撃者の視点で思考。** 回避技術、隠されたプロセス、改ざんされたシステムバイナリ、見落としがちな攻撃パターンを考慮するよう AI に促します。
- **並列実行。** 独立したチェックは同時に実行され、トリアージを高速化します。

これにより、このスキルは Debian、Ubuntu、RHEL、Alpine、Arch、その他あらゆる Linux ディストリビューションで修正なしに動作します。

## 要件

- カスタムスキルをサポートする AI コーディングアシスタント
- Linux サーバー（任意のディストリビューション）
- 対象サーバーでの root または sudo アクセス

## 実戦でテスト済み

このスキルは、Next.js の RCE 脆弱性を通じて侵害された本番サーバーでの実際のインシデントレスポンス中に開発・改良されました。調査で明らかになったもの：

- Monero マイニングプールを標的とした複数の XMRig マイナーの展開
- IoT ボットネットバイナリ（Mirai/Gafgyt 亜種）
- netcat 経由でペイロードを配信する 3 台のローテーション C2 サーバー
- シェルプロファイルインジェクションと cron ジョブによる永続化
- /tmp、/dev/shm、アプリケーションディレクトリに展開された UPX パック ELF バイナリ

このスキルの設計は、インシデント対応中に 3 回のイテレーションを経て、固定的なコマンドチェックリストから、現在の適応型・目標指向型フレームワークへと進化しました。
